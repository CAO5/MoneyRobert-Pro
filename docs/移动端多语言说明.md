# MoneyRobert 移动端多语言说明

当前支持：

- 简体中文 `zh-CN`
- 繁體中文 `zh-TW`
- English `en-US`
- 跟随系统

## 实现位置

- 翻译、语言定义与插值：`mobile/src/i18n/index.ts`
- 语言状态、系统语言识别、持久化和 TabBar 更新：`mobile/src/store/language.ts`
- 应用启动初始化：`mobile/src/app.tsx`
- 用户入口：我的 → 设置 → 语言

语言偏好保存在本地键 `mr_language_preference`。选择“跟随系统”时，应用启动会读取设备语言；无法识别的非中文语言默认使用英文。

## 开发约定

页面组件通过以下方式获取翻译：

```tsx
const { t, locale } = useI18n();

<Text>{t('今日决策')}</Text>
<Text>{t('你好，{name}', { name: username })}</Text>
```

日期和数字应使用当前 `locale`：

```tsx
new Date(value).toLocaleDateString(locale)
```

原生导航标题通过 `useLocalizedTitle` 动态更新：

```tsx
useLocalizedTitle('设置');
```

后端返回的业务文案不应直接返回中文句子。正式接口建议返回：

```json
{
  "message_key": "risk.drawdown_near_limit",
  "message_params": {
    "strategy": "S-2024-018",
    "drawdown": "7.2%"
  }
}
```

前端根据 `message_key` 本地化；未知 key 回退到后端提供的 `fallback_text`。

## 新增语言

1. 在 `AppLocale` 增加语言代码。
2. 新增对应语言包。
3. 在 `languageOptions` 增加设置选项。
4. 为日期、货币、涨跌色和文本方向补充区域规则。
5. 完成核心交易术语人工校对，不使用未经审核的机器翻译直接上线。

繁体中文目前基于简体文案进行字符转换。正式进入港澳台市场前，应建立独立 `zh-TW` 语言包，对“标的、持仓、回测、风控”等金融术语进行地区化校对。

## 分析报告语言一致性

- 所有移动端 API 请求自动携带当前 `Accept-Language`。
- 综合 AI 分析提示词强制要求叙述字段使用当前语言，JSON 协议字段保持稳定。
- 新生成报告在 `content.translations` 中保存 `zh-CN`、`zh-TW`、`en-US` 三个正文版本。
- 报告详情请求同时传递 `locale`，后端只选择完全匹配的标题和正文。
- 用户在报告页面停留期间切换语言，页面会重新获取对应版本，并通过请求序号避免旧请求覆盖新语言内容。
- 历史报告如果没有当前语言版本，接口返回 `language_match: false` 且不返回其他语言正文；客户端提示重新生成，禁止静默混用语言。
