# MoneyRobert 移动端敏感信息安全审计

## 已落实

- 账号密码只允许通过 HTTPS/TLS 请求体传输，不进入 URL、Storage 或日志。
- H5 仅允许 HTTPS 同源 API；localhost 页面仅用于本机开发。
- 小程序及其他移动端必须配置 `https://` API 绝对地址，否则请求在客户端发送前被阻断。
- 密码提交后立即从表单状态清除，登录页尝试启用平台截屏/录屏隐藏能力。
- Access Token、Refresh Token、用户资料仅保存在当前进程内存，应用退出后失效。
- 启动时删除旧版本写入 Taro Storage 的明文 Token 和用户资料。
- 所有移动端控制台日志已移除；HTTP 错误不再保存原始响应对象。
- `password`、`token`、`secret`、`api_key` 等敏感字段禁止进入 API 查询参数和页面跳转参数。
- 后端密码使用 bcrypt 哈希保存，不以明文或可逆密文入库。

## 密码保护边界

不在客户端内置对称密钥或固定 RSA 私钥。移动端安装包可以被逆向，内置密钥只能制造“看似加密”的假安全，还可能破坏服务端密码哈希和重放防护。

密码保护链路为：

1. 系统 TLS 验证服务器证书并加密整条传输通道。
2. 密码只存在于短暂请求体内。
3. 服务端通过 bcrypt 加盐哈希验证。
4. 登录成功后客户端仅持有短期会话 Token，且不落盘。

## 发布前硬性条件

微信小程序构建时必须提供正式 HTTPS 地址：

```powershell
$env:TARO_APP_API_URL = 'https://api.example.com/api/v1'
npm run build:weapp
```

当前仓库 `docker/nginx.conf` 和 `docker-compose.prod.yml` 仍只暴露 80 端口，没有包含证书配置。正式发布前必须在 CDN、负载均衡器或 Nginx 上完成 TLS 终止、80 → 443 跳转与 HSTS；否则新版移动端会主动拒绝登录和所有鉴权请求。

平台截图防护受操作系统和小程序能力限制，不能替代 TLS、最小化存储及设备锁。H5 浏览器无法可靠禁止系统截图。
