# MoneyRobert 移动端产品重设计与商业化方案

> **收费更新：** 原“10% 已实现净盈利服务费”方案已经废止。产品改为无强制费用、与收益金额无关的固定金额自愿赞助，具体以[《MoneyRobert 自愿赞助模式与监管边界》](./MoneyRobert自愿赞助模式与监管边界.md)为准。

## 1. 产品重新定位

**一句话定位：** 每天只告诉交易者最值得关注的一件事，并解释依据、风险和下一步。

MoneyRobert 不应是桌面量化后台的缩小版，也不应与交易所比拼盘口和下单。移动端的优势是高频触达、快速判断和即时风险响应，因此核心闭环是：

`发现变化 → 理解原因 → 形成决策 → 设置风控 → 交易后复盘 → 沉淀个人规则`

目标用户优先级：

1. 有真实持仓、但缺少系统分析能力的进阶个人交易者。
2. 同时关注股票、加密资产，信息源分散的跨市场用户。
3. 已使用量化策略，希望在移动端处理异常和复盘的专业用户。

暂不优先服务纯行情浏览用户和零基础“荐股”用户，避免产品落入低留存、强合规风险的流量模式。

## 2. 当前体验问题

| 问题 | 用户损失 | 重设计原则 |
| --- | --- | --- |
| “工作台/业务/待办”是内部系统语言 | 不知道先点哪里 | 改成首页、行情、策略、消息、我的 |
| 首页平铺指标与功能入口 | 信息很多，但没有结论 | 首屏只回答市场、资产、行动三件事 |
| 决策卡缺少时效和失效条件 | 容易把分析误解为买卖指令 | 展示置信度、有效期、关键价位和失效条件 |
| 风险告警藏在普通消息里 | 错过高价值触达 | 风险独立分级，必须有处理结果 |
| 标的详情仍为占位页 | 核心分析链路中断 | 建立“价格—信号—依据—风控—复盘”详情 |
| 报告要求回桌面端查看 | 移动价值被削弱 | 移动端提供结论版，桌面端承接深度研究 |
| 订阅收费与用户结果不一致 | 系统亏损时仍付费，信任成本高 | 只有系统交易已实现净盈利后才收费 |

## 3. 新信息架构

### 首页：每日决策台

- 市场脉搏：当前交易时段、整体波动和一句话环境判断。
- 资产卡：总资产、今日盈亏、短周期走势、账户同步状态。
- 今日唯一决策：系统按“影响 × 置信度 × 时效性”只推荐一条。
- 待处理风险：必须确认的风险、信号失效和数据异常。
- 我的关注：价格变化与信号状态并列展示。
- 价值回顾：本周预警、机会、复盘改进和保护价值估算。

### 行情：发现机会

- 顶部统一搜索，支持代码、名称和自然语言。
- 自选、机会榜、异动、市场四个视图。
- 排序不只看涨跌幅，增加“信号变化”“风险收益比”“与持仓相关性”。
- 标的详情包含行情、AI 解读、关键价位、相关持仓、历史信号表现。

### 策略：执行与复盘

- 我的策略：运行状态、收益、回撤、信号是否仍有效。
- 风险任务：从原“待办”升级为可执行任务，每条说明不处理的影响。
- 回测实验室：模板化参数、样本外验证、过拟合提示。
- 交易复盘：自动匹配买卖记录，识别追涨、扛亏、频繁交易等行为。
- 个人规则库：把复盘结论一键保存为下次交易前检查项。

### 消息：按决策价值分层

- 紧急风险：强提醒，有确认状态。
- 决策变化：关注标的的方向、置信度或失效条件变化。
- 任务结果：回测完成、报告生成。
- 产品通知：默认低优先级，不占用红点。

### 我的：账户与价值

- 策略授权状态、高水位线、待结算净利润和历史账单。
- 数据源与券商/交易所连接。
- 风险偏好、通知时段、隐私和安全。
- 分析方法说明、历史命中统计、免责声明。

## 4. 应有功能与待增加功能

### P0：形成可用闭环

- 多账户资产聚合与持仓同步。
- 每日决策、完整依据、置信度、有效期和失效条件。
- 自选信号变化与分级风险提醒。
- 标的分析详情，不再使用占位页。
- 交易导入、基础复盘和个人交易日志。
- 系统订单归因、高水位线账本、盈利结算和账单异议。
- 全链路埋点与实验分组。

### P1：形成留存壁垒

- 个性化“机会排序”，结合持仓、风险偏好和历史行为。
- 组合层风险：集中度、相关性、极端情景压力测试。
- AI 复盘教练与个人错误模式。
- 自然语言研究：“如果 BTC 跌破 65,000，我的组合会怎样？”
- 信号历史表现与可解释评估，区分训练内、样本外和实盘。
- 周报分享卡，隐藏金额后可安全分享。

### P2：形成增长与网络效应

- 可验证的策略模板市场，禁止只展示收益率。
- 研究员/策略作者主页，展示回撤、稳定性和实盘时长。
- 小组组合与投研协作。
- API/Webhook、桌面端高级策略编辑器。
- 企业风控台和团队权限。

## 5. 收费设计：只有系统交易盈利后才收费

### 唯一收费原则

- 不收订阅费、分析费、席位费和预付费。
- 只有带 MoneyRobert 策略 ID、由系统实际发出的订单，在平仓后产生可核验净盈利，才允许计费。
- 浮盈、手动交易盈利、外部策略盈利、入金、空投和汇率变化均不计费。
- 系统交易亏损时费用为 0；历史亏损没有完全弥补前，后续盈利也不收费。

### 建议费率与公式

首期验证采用**可计费净盈利的 10%**，不设置最低消费。

`可计费净盈利 = 系统已平仓盈利 - 系统已平仓亏损 - 手续费 - 税费 - 资金费 - 可核验滑点 - 待弥补历史亏损`

示例：本期系统盈利 ¥2,000、亏损 ¥500、成本 ¥100，且没有历史待弥补亏损，则可计费净盈利为 ¥1,400，服务费为 ¥140。若仍有 ¥800 历史亏损，则只对剩余 ¥600 计费，服务费为 ¥60。

### 高水位线

- 每个授权账户建立独立高水位线，首次值为策略启用时经过资金流归一化的账户净值。
- 每次结算后，仅当系统交易累计净利润创造新高，才对超过旧高水位线的部分收费。
- 入金与出金同步调整高水位线，不能把用户新增本金识别成盈利。
- 用户停止再重新启用策略时，历史亏损不能被重置。
- 多策略共享账户时采用账户级高水位线，避免一个策略盈利、另一个策略亏损却仍然收费。

### 归因与结算

- 系统订单必须记录 `strategy_id`、`signal_id`、`client_order_id`、成交、费用和撤改单链路。
- 用户手动改价、加仓或部分平仓后，该订单进入“混合归因”，默认不计费，直至用户确认归因规则。
- 每月最后一个自然日虚拟结算，生成逐笔可复核账单；只有用户确认后才扣费。
- 交易所冲正、退款或成交修正需自动回滚账单，并在下期抵扣。
- 用户可以随时停止策略、导出交易明细和提出结算异议。

### 收费前置条件

1. 接入具备交易与成交回报能力的券商或交易所，而不是依赖用户手工记录。
2. 系统具备订单执行、风控熔断、撤单和异常恢复能力。
3. 完成投资者适当性评估、书面交易授权、风险揭示与服务协议。
4. 通过持牌主体或持牌合作方提供服务，确认所在地允许自动交易和收益分成。
5. 结算资金不经过个人账户，收费主体、标准、发票与投诉渠道保持一致。

### 合规边界

在中国大陆，面向个人客户代为决定并执行证券交易、再约定分享收益，存在较高的无牌投资咨询、违规代客理财和业绩报酬风险。产品原型可以实现账本和结算展示，但在持牌路径、合作券商与专项法律意见完成前，必须关闭真实自动交易和扣费能力。

若采用私募基金或持牌资管产品路径，业绩报酬需遵循基金合同、信息披露、适当性和高水位线等规则，不能把普通软件服务协议直接包装成收益分成。

### 商业盈利模型

假设平均授权账户资金为 ¥100,000，系统年度已实现净收益为 8%，收费比例为 10%：

- 单个盈利账户理论年收入：`100,000 × 8% × 10% = ¥800`。
- 按 50% 账户年度盈利、支付与合作方综合成本 25% 估算，全部授权账户平均收入约 ¥300。
- 若单账户年度行情、AI、交易和客服成本为 ¥100，则平均贡献毛利约 ¥200。
- 获客成本必须控制在 ¥120 以下，并维持足够的策略授权留存。

以上仅是商业敏感性测算，不是收益预测。若系统长期不能产生经审计的正向实盘结果，该收费模式不会形成收入，因此上线前必须完成足够长的影子盘和小规模实盘验证。

## 6. 关键交互规则

- 涨跌数据必须使用等宽数字，金额默认可隐藏。
- “买入/卖出”改为“关注/观察/减仓/规避”等分析措辞，并始终展示失效条件。
- 红色只用于真实风险和 A 股上涨语义，避免普通按钮滥用红色。
- 所有风险提醒必须支持“已处理、稍后提醒、与我无关”。
- 下拉刷新保留上次数据和时间戳，不使用整页空白加载。
- 空状态给出下一步动作，错误状态保留重试和离线缓存。
- 付费能力不可伪装成系统错误；锁定内容明确展示权益和替代免费能力。

## 7. 指标体系

北极星指标：**每周完成有效决策闭环的用户数**。

“有效决策闭环”定义为用户完成以下任意一条：

- 查看完整决策依据并设置提醒。
- 处理风险任务并记录结果。
- 导入交易并完成一次复盘。
- 根据复盘结论建立个人规则。

核心漏斗：

`连接账户 → 完成适当性 → 授权策略 → 系统执行交易 → 形成已实现净盈利 → 用户核对账单 → 收取盈利服务费`

守护指标：

- 风险提醒误报率与关闭通知率。
- 决策详情阅读完成率。
- 7 日/30 日留存。
- 策略授权率、盈利账户率、账单确认率、停止策略率。
- 单活跃用户计算成本和单付费用户毛利。
- 投诉率、风险措辞合规命中率。

## 8. 12 周落地路线

### 第 1–4 周：把核心链路做通

- 上线新首页、标的详情、风险任务和只读盈利结算账本。
- 接入 Mobile BFF 聚合接口与埋点。
- 让 20–30 名种子用户完成真实账户连接和一周使用。

### 第 5–8 周：证明留存

- 上线交易导入、AI 复盘、个人规则。
- 优化风险误报和决策解释。
- 先以影子盘验证订单归因、高水位线和风控，不开放真实扣费。

### 第 9–12 周：验证商业化

- 在持牌合作路径完成后，小规模验证 10% 盈利服务费。
- 上线年度价值报告和安全分享。
- 以贡献毛利、留存和退订率共同决定是否扩大投放。

## 9. 后端与数据改造清单

- `GET /mobile/home`：市场状态、资产摘要、今日决策、风险、自选和价值证明的一次聚合响应。
- `GET /symbols/{symbol}/analysis`：标的完整移动分析。
- `POST /alerts/{id}/actions`：风险处理结果。
- `GET /portfolio/risk`：组合相关性、集中度和压力测试。
- `POST /trades/import`、`GET /reviews`：交易导入与复盘。
- `GET /settlements/preview`、`POST /settlements/{id}/confirm`：虚拟结算、逐笔明细与用户确认。
- `GET /accounts/{id}/high-water-mark`：账户级高水位线与资金流调整记录。
- 价值事件：`strategy_authorized`、`system_order_filled`、`position_closed`、`settlement_previewed`、`settlement_confirmed`、`settlement_disputed`。

价值估算只可作为解释性指标，必须展示口径，不得暗示真实收益或保证结果。
