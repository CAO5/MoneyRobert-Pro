## 第 1 章 启动 AI 综合分析

### 1.1 AI 分析页面概述

访问 `http://localhost:3000/ai`，有 4 个 Tab：

| Tab | 名称 | 用途 |
|-----|------|------|
| 技术分析 | 单维度技术面分析 | 仅基于 K 线、RSI、MACD 等技术指标 |
| 资金分析 | 单维度资金面分析 | 仅基于资金费率、持仓量、多空比 |
| 情绪分析 | 单维度情绪面分析 | 仅基于新闻舆情 |
| 综合分析 | 三维度综合 | 技术面 + 资金面 + 情绪面 → 综合建议 |

⚠️ AI 综合分析**不是多 Agent 辩论**，它是一次 LLM 调用得到结果，速度快但深度不足。完整决策应使用第 2 章的多 Agent 辩论。

### 1.2 使用方式

1. 顶部选择交易对（如 `BTC-USDT-SWAP`）
2. 切换到【综合分析】Tab
3. 点击"开始分析"
4. 等待 10-30 秒返回结果

### 1.3 结果解读

| 字段 | 含义 | 注意点 |
|------|------|--------|
| 方向 | long / short / hold | hold 时不交易 |
| 置信度 | 0-1 | < 0.6 建议观望 |
| 风险等级 | low / medium / high | high 建议不交易 |
| 杠杆建议 | 1-5x | 受 max_leverage 上限约束 |
| 入场区间 | 价格范围 | 用于挂限价单 |
| 止损价 | 触发止损的价格 | **必须设置** |
| 止盈价 | 触发止盈的价格 | 推荐分级止盈 |
| 分析理由 | LLM 给出的判断依据 | 用于事后复盘 |

### 1.4 API 调用方式

```bash
TOKEN="你的_access_token"

curl -X POST http://localhost:8001/api/v1/ai/analyze/comprehensive \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"symbol": "BTC-USDT-SWAP"}'
```

## 第 2 章 启动多 Agent 辩论

### 2.1 多 Agent 辩论 vs AI 综合分析

| 维度 | AI 综合分析 | 多 Agent 辩论 |
|------|-----------|-------------|
| 调用次数 | 1 次 LLM | 多次 LLM（每 Agent 1-2 次 + 跨部门辩论） |
| 耗时 | 10-30 秒 | 30-90 秒 |
| 深度 | 浅 | 深（多角度 + 互相质疑） |
| 成本 | 低 | 高（Token 用量大） |
| 用途 | 快速判断 | 最终决策依据 |

⚠️ **paper 模式自动交易内部使用的是多 Agent 辩论**，不是 AI 综合分析。

### 2.2 13 类 Agent 介绍

辩论由 4 个部门共 13 个分析师 Agent + 1 个基金经理 Agent 组成：

#### 技术部 Technical（权重 0.35）

| Agent | 中文名 | 关注信号 |
|-------|-------|---------|
| KlinePatternAnalyst | K 线形态分析师 | 价格变动、24h 涨跌幅、高低位 |
| TechnicalIndicatorAnalyst | 技术指标分析师 | RSI_14、MACD 信号 |
| OnChainDataAnalyst | 链上数据分析员 | 持仓量 OI、24h 成交量 |
| QuantModelAnalyst | 量化模型分析师 | 24h 波动率、多因子模型 |

#### 资金部 Capital（权重 0.35）

| Agent | 中文名 | 关注信号 |
|-------|-------|---------|
| FundingRateAnalyst | 资金费率分析师 | funding_rate（负 = 偏空，正 = 偏多） |
| PositionStructureAnalyst | 持仓结构分析师 | OI 持仓结构变化 |
| LongShortGameAnalyst | 多空博弈分析师 | long_short_ratio |
| LiquidityAnalyst | 流动性分析师 | 24h 成交量 |

#### 新闻部 News（权重 0.30）

| Agent | 中文名 | 关注信号 |
|-------|-------|---------|
| SentimentAnalyst | 舆情情绪分析师 | 新闻平均 sentiment |
| MacroPolicyAnalyst | 宏观政策分析师 | 宏观新闻、监管动态 |
| KOLWhaleMonitor | KOL/鲸鱼监控员 | KOL 言论、鲸鱼动作 |
| EventDrivenAnalyst | 事件驱动分析师 | 黑天鹅、利好利空事件 |

#### 基金经理 FundManager

- **FundManagerAgent**：综合所有分析师观点 + 历史决策参考 + 风险评估 → 最终决策

### 2.3 辩论四阶段流程

```
Phase 1: 独立分析
  ↓ 12 个分析师各自独立给出观点（sentiment + confidence + reasoning）
Phase 2: 部门内辩论
  ↓ 同部门 sentiment 不同的 Agent 互相辩论、修正观点
Phase 3: 跨部门辩论
  ↓ 看多阵营 vs 看空阵营，跨部门对峙
Phase 4: 基金经理最终决策
  ↓ 综合所有辩论结果 + 动态可信度 + 风险否决 → FundManagerDecision
```

### 2.4 启动辩论

#### 方式 A：前端

访问 `http://localhost:3000/ai`（或专门的辩论查看器 `http://localhost:3000/ai/debate`，如果有的话）：

1. 选择交易对
2. 点击"启动辩论"
3. 等待 30-90 秒
4. 实时查看各 Agent 发言

#### 方式 B：API

```bash
curl -X POST http://localhost:8001/api/v1/agent/debate/start \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "symbol": "BTC-USDT-SWAP",
    "config_id": "可选,关联模拟盘配置"
  }'
```

返回：

```json
{
  "session_id": "uuid-xxxx-xxxx-xxxx",
  "status": "started"
}
```

用 `session_id` 查询辩论详情：

```bash
curl http://localhost:8001/api/v1/agent/debate/{session_id} \
  -H "Authorization: Bearer $TOKEN"
```

## 第 3 章 解读辩论结果

辩论结束返回 `FundManagerDecision` 对象（[backend/src/agents/models.rs:65-79](file:///d:/wan_workspase/MoneyRobert-Pro/backend/src/agents/models.rs)）。

### 3.1 核心字段详解

```json
{
  "session_id": "uuid-xxxx",
  "action": "long",                    // long / short / hold
  "symbol": "BTC-USDT-SWAP",
  "confidence": 0.72,                  // 0-1
  "position_size_percent": 3.5,        // 建议仓位占比
  "leverage": 2,                       // 建议杠杆
  "stop_loss_percent": -2.0,           // 止损百分比（负数）
  "take_profit_percent": [1.5, 2.5, 4.0],  // 分级止盈
  "reasoning": "技术面 K 线突破...",    // 决策理由
  "agent_contributions": [...],        // 各 Agent 贡献
  "risk_assessment": {
    "overall_risk_level": "medium",    // low / medium / high
    "max_position_risk": 0.018,
    "margin_requirement": 350,
    "risk_reward_ratio": 2.0,
    "volatility_rating": "normal",
    "alerts": []
  },
  "timestamp": "2026-06-25T12:00:00Z"
}
```

### 3.2 action 字段决策表

| action | 含义 | 执行动作 |
|--------|------|---------|
| `long` | 做多 | 开多仓 |
| `short` | 做空 | 开空仓 |
| `hold` | 观望 | **不交易**，等待下次辩论 |

⚠️ **新手常见错误**：看到 `hold` 就降低阈值强行交易。这是大忌。`hold` 就是当前没机会，应该等待。

### 3.3 confidence 字段判断

| confidence 范围 | 含义 | 建议 |
|---------------|------|------|
| ≥ 0.75 | 高置信度 | ✅ 可执行（仍需通过其他门禁） |
| 0.65 - 0.75 | 中等置信度 | ⚠️ 谨慎，可减半仓位 |
| 0.55 - 0.65 | 低置信度 | ❌ 建议观望 |
| < 0.55 | 极低 | ❌ 不交易 |

**默认阈值**：`ai_confidence_threshold = 0.7`（数据库默认，可在设置页改）。低于此值自动跳过。

⚠️ **paper 模式特殊**：paper 模式自动把阈值 × 0.7（即 0.7 × 0.7 = 0.49），便于积累样本。**demo 和 live 模式不降低阈值**。

### 3.4 risk_assessment 字段判断

| 字段 | 含义 | 危险信号 |
|------|------|---------|
| `overall_risk_level` | 整体风险等级 | = `high` 时不交易 |
| `max_position_risk` | 最大仓位风险占比 | > 0.02（2%）时不交易 |
| `risk_reward_ratio` | 风险收益比 | < 1.5 时不交易 |
| `volatility_rating` | 波动率评级 | = `extreme` 时不交易 |
| `alerts` | 风险警报列表 | 非空时谨慎 |

### 3.5 agent_contributions 字段

每个分析师的贡献：

```json
{
  "agent_name": "tech_kline_analyst",
  "department": "Technical",
  "sentiment": "Bullish",          // Bullish / Bearish / Neutral / Cautious
  "confidence": 0.65,
  "contribution_weight": 0.12,     // 该 Agent 对最终决策的权重
  "credibility_score": 0.72       // 该 Agent 的可信度（基于历史表现）
}
```

**分歧识别**：如果 Bullish 和 Bearish 的 Agent 数量接近（如 6:6），说明市场分歧大，建议降低仓位或观望。

## 第 4 章 生成概率决策卡

### 4.1 决策卡是什么

决策卡把模糊的"AI 觉得会涨"转化为**可审计的概率对象**：

```
不输出："BTC 会涨到 11 万"
输出："BTC 1 小时内上涨概率 45%，下跌 35%，震荡 20%，
       净期望 EV = +0.8%，CVaR_95 = -2.1%，失效条件：跌破 95000"
```

### 4.2 决策卡页面

访问 `http://localhost:3000/signals/decision-card`：

- **左侧**：生成表单
- **右上**：最新决策卡可视化
- **右下**：历史决策卡列表

### 4.3 生成决策卡步骤

#### 步骤 1：填写交易对与预测周期

- 交易对：`BTC-USDT-SWAP`
- 预测周期：可选 15 分钟 / 1 小时 / 4 小时 / 1 天
- 新手建议：1 小时起步

#### 步骤 2：填写概率分布

让 p_up + p_down + p_flat = 1.0：

- p_up：上涨概率（如 0.45）
- p_down：下跌概率（如 0.35）
- p_flat：震荡概率（自动计算 = 1 - p_up - p_down = 0.20）

底部进度条会显示三段比例，颜色：绿色=上涨，灰色=震荡，红色=下跌。

#### 步骤 3：填写收益分位数

- Q10（悲观）：10% 概率会跌到的最差收益（如 -0.02，即 -2%）
- Q50（中位）：50% 概率的中位收益（如 0.005，即 +0.5%）
- Q90（乐观）：10% 概率会涨到的最好收益（如 0.03，即 +3%）

要求：Q10 < Q50 < Q90

#### 步骤 4：填写 EV 与风险

- **净期望 EV**：扣除手续费、滑点、资金费率后的期望收益（如 0.008）
  - ⚠️ 必须 > 0，否则就是负期望交易，长期必亏
- **仓位建议**：0-1 之间（如 0.05 = 5%）
- **最坏情形 CVaR**：95% 置信下最坏情形平均损失（如 -0.025，即 -2.5%）
- **已用风险预算**：0-1 之间（如 0.3 = 30%）

#### 步骤 5：填写模型版本与市场状态

- 模型版本：`v1.0.0`
- 市场状态：trending / ranging / volatile / breakout / unknown

#### 步骤 6：填写数据新鲜度

- 数据新鲜度（秒）：建议 < 300（如 120）

#### 步骤 7：填写信任闭环字段（v1.8 新增，强烈建议）

- **决策原因**（逗号分隔）：`资金费率为负,技术面突破,辩论置信度0.72`
- **阻断原因**（逗号分隔）：`宏观不确定,数据新鲜度差`
- **可信等级**下拉框：
  - `display_only`：仅展示，不能作为晋级依据
  - `comparable`：可用于策略间比较
  - `promotion_eligible`：达到晋级实盘的可信度

#### 步骤 8：点击"生成决策卡"

成功后右侧显示最新决策卡，底部历史列表更新。

### 4.4 决策卡判定标准（是否值得执行）

一笔交易必须**同时**满足以下 5 个条件才执行：

```
✅ EV > 0.005（0.5%）
✅ P(EV > 0) > 0.6
✅ CVaR_95 < 风险预算（≤ 2% 账户权益）
✅ data_freshness_sec < 300
✅ trust_level != display_only（用于实盘时）
```

任一条件不满足 → **观望**。

### 4.5 API 调用方式

```bash
curl -X POST http://localhost:8001/api/v1/signals/decision-card \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{

    "target_horizon_sec": 3600,

    "q10": -0.02,
    "q50": 0.005,

```

## 第 5 章 创建并执行回测

### 5.1 回测的目的

回测是用历史数据验证策略在过去的表现，回答：

- 这个策略在过去 6 个月赚了还是亏了？
- 最大回撤多少？
- 胜率多少？盈亏比多少？
- **可信等级**：是仅展示（display_only）、可比较（comparable）、还是可晋级（promotion_eligible）？

⚠️ **回测可信等级是后续晋级 demo/live 的硬性门禁**。

### 5.2 回测中心页面

访问 `http://localhost:3000/backtest`：

- **左侧**：回测任务列表（状态徽章、收益、Sharpe）
- **右侧**：选中任务详情
  - 任务配置（标的、时间、费用、滑点、杠杆）
  - 绩效报告（总收益、Sharpe、回撤、胜率等 8 项）
  - **可信等级门禁**（6 项检查、晋级资格、评估指标）

### 5.3 创建回测任务

#### 前端步骤

1. 点击左侧"创建回测任务"
2. 填入：
   - 任务名：`BTC trend v1`
   - 交易对：`BTC-USDT-SWAP`
   - 起始时间：`2025-01-01`
   - 结束时间：`2025-06-01`（建议 ≥ 6 个月）
   - 初始资金：`10000`
   - 手续费率 taker：`0.0005`（5 bp，永续合约标准）
   - 手续费率 maker：`0.0002`（2 bp，挂单）
   - 滑点 bps：`5`（5 bp）
   - 杠杆：`2`
   - 日亏损上限：`5`
3. 点击"创建"
4. 在任务列表点击"启动"

#### API 方式

```bash
# 创建任务
curl -X POST http://localhost:8001/api/v1/backtest/jobs \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "job_name": "BTC trend v1",
    "assets": ["BTC-USDT-SWAP"],
    "start_time": "2025-01-01T00:00:00Z",
    "end_time": "2025-06-01T00:00:00Z",
    "initial_equity": 10000,
    "fee_taker_bps": 5,
    "fee_maker_bps": 2,
    "slippage_bps": 5,
    "max_single_position_pct": 0.05,
    "max_total_leverage": 2,
    "max_daily_loss_pct": 5
  }'

# 返回 job_id 后启动
curl -X POST http://localhost:8001/api/v1/backtest/jobs/{job_id}/start \
  -H "Authorization: Bearer $TOKEN"
```

### 5.4 等待回测完成

回测耗时取决于：

- 数据量（时间跨度 × 交易对数）
- 策略复杂度（信号触发频率）
- 系统负载

5 分钟 K 线 × 6 个月 × 单交易对，通常 1-5 分钟完成。

可通过 API 查询进度：

```bash
curl http://localhost:8001/api/v1/backtest/jobs/{job_id} \
  -H "Authorization: Bearer $TOKEN"
```

返回包含 `status`（pending/running/completed/failed）和 `progress`（0-100）。

### 5.5 查看绩效报告

完成后点击任务，右侧显示绩效报告：

| 指标 | 含义 | 健康值 |
|------|------|--------|
| 总收益（净） | 扣费后总收益 % | > 0 |
| 年化收益 | 年化后的收益率 | > 15% |
| Sharpe 比率 | 风险调整后收益 | > 1.0 |
| 最大回撤 | 历史最大亏损幅度 | < 20% |
| 胜率 | 盈利交易占比 | > 50% |
| 盈亏比 | 平均盈利 / 平均亏损 | > 1.5 |
| 总手续费 | 累计手续费金额 | — |
| 平均持仓时长 | 每笔交易持仓时间 | — |

### 5.6 查看可信等级门禁

右侧"可信等级门禁"区域显示 6 项检查：

| 门禁 | 含义 | 通过条件 |
|------|------|---------|
| 测试覆盖 | 单元测试是否齐全 | 见 03 篇 |
| 资金守恒 | 任意成交序列权益守恒 | 无负数量、无重复记账 |
| 滑点入账 | 滑点成本是否计入 | slippage_cost 字段非 0 |
| 样本量 | 交易笔数 | ≥ 30 |
| Walk-forward | 训练/验证/测试切分 | 见 03 篇 |
| 校准健康 | Brier Score 校准 | < 0.33 |

最终等级：

| 等级 | 含义 | 可执行操作 |
|------|------|---------|
| `display_only` | 仅展示 | ❌ 不可作为晋级依据 |
| `comparable` | 可比较 | ⚠️ 可用于策略对比 |
| `promotion_eligible` | 可晋级 | ✅ 可作为晋级 demo/live 依据 |

API 查询：

```bash

  -H "Authorization: Bearer $TOKEN"
```

⚠️ **新手铁律**：永远不要把 `display_only` 的策略直接上 demo 或 live。这是防"过拟合回测必爆仓"的核心保护。

### 6.4 自动交易循环逻辑（务必理解）

启动后系统自动运行 `spawn_simulation_loop`（[backend/src/routes/agent_simulation.rs:717](file:///d:/wan_workspase/MoneyRobert-Pro/backend/src/routes/agent_simulation.rs)），每 5 分钟一次循环：

```
循环开始：
  1. 检查 config.status == "running"（否则退出循环）
  2. 加载最新 config（资金、参数）
  3. 平仓阶段：
     - 查询所有 status="open" 的交易
     - 对每笔交易检查：
       - 止损触发？ → 平仓（reason="stop_loss"）
       - 止盈触发？ → 平仓（reason="take_profit"）
       - 持仓 > 240 分钟？ → 平仓（reason="timeout"）
     - 如果有平仓 → 立即重新辩论开新仓（闭环）
  4. 开仓阶段：
     - 检查日交易数 < max_daily_trades
     - 检查当前持仓数 == 0
     - 满足 → 调用多 Agent 辩论
     - action == "hold" → 跳过
     - confidence < 阈值 → 跳过（paper 模式阈值 × 0.7）
     - 计算 position_value = current_balance × (max_position_size_percent / 100)
     - 计算 quantity = position_value / current_price
     - leverage = min(debate.leverage, max_leverage)
     - 设置 stop_loss, take_profit
     - 执行 execute_trade()
  5. 更新滚动统计（胜率、盈亏比、最大回撤、连续止损）
  6. 等待 5 分钟 → 下一次循环
```

