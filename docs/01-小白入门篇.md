## 第 1 章 认识 MoneyRobert Pro

### 1.1 这是什么系统

MoneyRobert Pro 是一个 **"AI 辅助交易工作台 + 确定性风控执行系统"**，专为加密永续合约交易设计。

它不是"印钞机"，而是把交易决策这件事**结构化**：

- AI 多 Agent 辩论 → 给出方向与置信度
- 概率决策卡 → 把模糊判断变成可审计的概率对象
- 回测可信等级门禁 → 防止过拟合策略上实盘
- 风控引擎 → 自动止损、限仓、断路

### 1.2 能做什么 / 不能做什么

| 能做 | 不能做 |
|------|--------|
| 采集 OKX 行情（K 线/资金费率/持仓量等） | 自动预测价格涨跌 |
| 启动 13 类 Agent 多边辩论 | 保证盈利（盈利取决于策略与纪律） |
| 生成概率决策卡（EV/CVaR/失效条件） | 替你做最终决策（最终决策权在人） |
| 跑回测并给出可信等级门禁 | 取代交易所（资金仍在 OKX） |
| Paper/Demo/Live 三模式自动交易 | 跨交易所套利（暂只对接 OKX） |
| 自动止损/止盈/超时平仓 | 自动提现/划转资金 |

### 1.3 系统的四个核心概念

#### 概念 1：行情数据是基础

系统每 5 秒从 OKX 拉取行情（K 线、Ticker、资金费率、持仓量等 8 类），存入数据库。**没有数据，后续一切免谈**。

#### 概念 2：多 Agent 辩论是大脑

13 个 Agent（K 线分析师、技术指标分析师、资金费率分析师、宏观分析师等）从不同角度分析，基金经理 Agent 综合所有观点给出最终决策。

#### 概念 3：概率决策卡是输出

不输出"会涨到多少"，而是输出：

- 上涨概率 45%，下跌概率 35%，震荡概率 20%
- 净期望 EV = +0.8%（扣费后）
- CVaR_95 = -2.1%（最坏情形）
- 失效条件：跌破 95000

#### 概念 4：风控是底线

不管 AI 多自信，必须通过：

- 单笔风险 ≤ 2% 账户
- 当日亏损 ≤ 5% 账户
- CVaR ≤ 风险预算
- 数据新鲜度 ≤ 300 秒

## 第 2 章 第一次部署

### 2.1 环境要求

| 软件 | 最低版本 | 用途 |
|------|---------|------|
| Docker Desktop | 24.x | 容器运行时 |
| Docker Compose | v2.x | 编排（Docker Desktop 已内置） |
| 浏览器 | Chrome / Edge 最新版 | 访问前端 |

> Windows 用户：在 Docker Desktop 设置中开启 **WSL2 引擎**（默认已开），并分配 ≥ 4GB 内存给 Docker。

### 2.2 一键启动

打开 PowerShell（管理员权限不是必须），切换到项目根目录：

```powershell
cd D:\wan_workspase\MoneyRobert-Pro
docker compose up -d
```

首次启动会拉取镜像 + 编译后端，可能耗时 5-15 分钟（取决于网速）。后续启动 < 30 秒。

启动后会有 4 个容器：

| 容器名 | 端口 | 用途 |
|--------|------|------|
| `moneyrobert-pro-backend` | 8001 | Rust Axum API 服务 |
| `moneyrobert-pro-frontend` | 3000 | Vue 3 前端（Vite dev 模式） |
| `moneyrobert-pro-db` | 5432 | PostgreSQL 15 + pgvector |
| `moneyrobert-pro-redis` | 6379 | Redis 7 缓存 |

### 2.3 验证部署成功

#### 方式 A：浏览器访问前端

打开 `http://localhost:3000`，看到登录页即成功。

#### 方式 B：API 健康检查

```powershell
curl http://localhost:8001/api/v1/
```

返回 `{"status":"ok",...}` 类似 JSON 即成功。

#### 方式 C：查看容器状态

```powershell
docker compose ps
```

4 个容器全部 `Up` 状态即成功。如果 `db` 或 `redis` 显示 `Restarting`，检查端口是否被占用。

### 2.4 常见部署问题

#### Q1：端口被占用

```
Error starting userland proxy: Bind for unexpected address 0.0.0.0:5432
```

修改 `docker-compose.yml`，把 `5432:5432` 改成 `5433:5432`（前面是宿主机端口）。

#### Q2：磁盘空间不足

```
no space left on device
```

清理 Docker 资源：

```powershell
docker system prune -a --volumes
```

⚠️ 这会删除所有未使用的镜像和卷，谨慎执行。

#### Q3：数据库迁移失败

查看 db 容器日志：

```powershell
docker logs moneyrobert-pro-db
```

如果看到 `permission denied` 或 `relation already exists`，删除卷重来：

```powershell
docker compose down -v
docker compose up -d
```

⚠️ `down -v` 会清空数据库所有数据，仅在首次部署时使用。

#### Q4：前端访问 404

前端容器是 Vite dev 模式，热更新需要源码挂载。检查 `docker-compose.yml` 的 `frontend.volumes` 是否正确挂载了 `./frontend/src`。

### 2.5 停止与重启

```powershell
# 停止全部服务（保留数据）
docker compose stop

# 启动已停止的服务
docker compose start

# 重启某个服务（修改配置后常用）
docker compose restart backend

# 完全停止并删除容器（数据保留在卷里）
docker compose down

# 重新启动
docker compose up -d
```

## 第 3 章 注册与登录

### 3.1 注册第一个账户

1. 浏览器打开 `http://localhost:3000/register`
2. 填入用户名、邮箱、密码（建议 ≥ 12 位，含大小写 + 数字 + 符号）
3. 点击"注册"

⚠️ 首次部署时第一个注册的账户通常拥有管理员权限。

### 3.2 登录系统

1. 浏览器打开 `http://localhost:3000/login`
2. 输入邮箱和密码
3. 点击"登录"
4. 登录成功后自动跳转到仪表盘 `http://localhost:3000/dashboard`

### 3.3 Token 机制（简单了解）

登录后系统返回两个 Token：

- **access_token**：30 分钟有效期，存入 `localStorage`
- **refresh_token**：7 天有效期，存入 `localStorage`

前端 axios 拦截器会自动：

- 每次请求带 `Authorization: Bearer <access_token>`
- access_token 过期时自动用 refresh_token 刷新
- refresh_token 也过期时跳转到登录页

**你不需要手动管理 Token**，浏览器保持登录态即可。

### 3.4 退出登录

点击右上角头像 → 退出登录，或访问 `http://localhost:3000/login`。

## 第 4 章 看懂仪表盘

仪表盘地址：`http://localhost:3000/dashboard`

### 4.1 仪表盘的 4 个核心卡片

| 卡片 | 含义 | 看什么 |
|------|------|--------|
| **总权益** | 当前账户总价值（USDT） | 新注册用户为 0，启动模拟盘后才有值 |
| **今日盈亏** | 当日已实现 + 未实现盈亏 | 正数绿色，负数红色 |
| **持仓数** | 当前未平仓的合约数量 | 模拟盘启动后会显示 |
| **活跃策略** | 运行中的自动交易配置数 | 0 表示无自动交易在运行 |

新注册用户这 4 个卡片都是 0，这是正常的——因为还没有启动模拟盘。

### 4.2 持仓摘要表

如果启动了模拟盘，下方会显示当前持仓列表：

- 交易对（如 BTC-USDT-SWAP）
- 方向（做多/做空）
- 数量
- 开仓价
- 当前价
- 未实现盈亏（USDT 与百分比）
- 止损价 / 止盈价

### 4.3 快捷操作区

仪表盘右侧通常会有快捷操作按钮：

- 启动 AI 分析 → 跳转 `/ai`
- 创建决策卡 → 跳转 `/signals/decision-card`
- 跑回测 → 跳转 `/backtest`

这些功能在 02 篇会详细介绍。

### 4.4 市场状态卡片（如果显示）

如果行情数据已采集，仪表盘顶部可能显示当前市场状态：

- **趋势**（trending_bull / trending_bear）
- **震荡**（ranging）
- **高波动**（high_volatility）
- **危机**（crisis）

这是系统根据 ADX、波动率分位等指标自动识别的市场状态。**危机状态下建议不交易**。

## 第 5 章 看懂行情页

行情页地址：`http://localhost:3000/market`

### 5.1 行情页的核心区域

| 区域 | 内容 |
|------|------|
| 顶部 | 交易对选择器（如 BTC-USDT-SWAP）+ 刷新按钮 |
| 市场状态卡片 | 当前 regime + 置信度 + ADX + 波动分位 |
| 数据质量卡片 | 质量等级 + 覆盖率 + 缺口率 + 新鲜度 |
| K 线图 | 7 个周期可选（1m / 5m / 15m / 30m / 1H / 4H / 1D） |
| 资金费率图 | 当前资金费率 + 历史曲线 |
| 多空比图 | 多头/空头持仓比例 |
| 持仓量图 | Open Interest 走势 |
| 行情列表 | 多个交易对的实时价格列表 |

### 5.2 市场状态卡片详解

| 状态 | 含义 | 交易建议 |
|------|------|---------|
| `trending_bull` | 趋势上涨 | 顺势做多 |
| `trending_bear` | 趋势下跌 | 顺势做空 |
| `ranging` | 震荡 | 观望或区间策略 |
| `high_volatility` | 高波动 | 减小仓位、加宽止损 |
| `crisis` | 危机 | **不交易** |

**置信度**：0-1 之间，越高表示状态识别越确定。< 0.5 时建议谨慎。

**ADX**：趋势强度指标，> 25 表示强趋势，< 20 表示震荡。

### 5.3 数据质量卡片详解

这是系统最关键的保护机制之一：

| 等级 | 含义 | 交易建议 |
|------|------|---------|
| 优秀 | 数据完整新鲜 | ✅ 可交易 |
| 良好 | 数据基本完整 | ✅ 可交易 |
| 一般 | 有缺口或延迟 | ⚠️ 谨慎 |
| 较差 | 数据严重缺失 | ❌ 不交易 |

**关键指标**：

- **覆盖率**（coverage_ratio）：0-1，越高越好，应 > 0.95
- **缺口率**（gap_ratio）：0-1，越低越好，应 < 0.05
- **新鲜度**（freshness_sec）：数据最后更新到现在的秒数，应 < 300

⚠️ **新手铁律**：新鲜度 > 1800 秒（30 分钟）绝对不要交易。

### 5.4 K 线图使用

- 顶部周期按钮切换 K 线周期（1m 实时性最强，1D 适合看趋势）
- 鼠标悬停 K 线显示 OHLCV 数据
- 可拖拽缩放查看历史

### 5.5 资金费率图

资金费率是永续合约的核心机制：

- **正资金费率**（如 +0.01%）：多头付钱给空头，市场偏多
- **负资金费率**（如 -0.01%）：空头付钱给多头，市场偏空
- 每 8 小时结算一次

**新手认知**：资金费率绝对值过大（> 0.05%）表示市场极端，建议谨慎。

### 5.6 行情列表

底部列表显示多个交易对的：

- 当前价格
- 24h 涨跌幅
- 24h 成交量
- 资金费率

点击某行可切换顶部主图显示该交易对。

## 第 6 章 看懂设置页

设置页地址：`http://localhost:3000/settings`

### 6.1 设置页的核心区域

| 区域 | 用途 |
|------|------|
| 决策调优台 | 调整多 Agent 辩论的部门权重、最低方向优势等（资深篇详讲） |
| 代理配置 | 配置 SOCKS5 / HTTP / HTTPS 代理访问 OpenAI 等 |
| API 密钥管理 | 录入 OKX 交易所 API Key、AI 提供商 API Key |
| AI 模型配置 | 配置 OpenAI / DeepSeek / Anthropic / 自定义模型 |

### 6.2 代理配置（国内用户必看）

如果在国内访问 OpenAI 等 AI 服务需要代理：

1. 在【代理配置】区域填入：
   - 类型：SOCKS5 / HTTP / HTTPS
   - 地址：如 `127.0.0.1`
   - 端口：如 `7890`
2. 点击"测试连接"
3. 显示"连接成功"后保存

⚠️ 不配置代理时，AI 分析功能会回退到 fallback 逻辑（基于规则生成观点），效果较弱。

### 6.3 API 密钥管理

#### 6.3.1 OKX 交易所 API Key

⚠️ **这一步是后续所有交易功能的前置条件**。

1. 登录 [OKX 官网](https://www.okx.com/)
2. 头像 → API 管理 → 创建 API Key
3. **强烈建议先创建 demo trading API Key**（OKX 模拟盘）
4. 权限勾选：`读取` + `交易`（**不要勾提现**）
5. 记录三个值：
   - `API Key`
   - `Secret Key`
   - `Passphrase`

回到设置页 → API 密钥管理 → 新增交易所密钥：

- 交易所：OKX
- API Key / Secret Key / Passphrase 填入
- **模拟盘标识：勾选**（重要！）

#### 6.3.2 AI 提供商 API Key

至少配置一个 AI 提供商，否则辩论功能使用 fallback：

- **DeepSeek**（国内可用，性价比高）：[获取 Key](https://platform.deepseek.com/)
- **OpenAI**（需代理）：[获取 Key](https://platform.openai.com/)
- **Anthropic Claude**（需代理）：[获取 Key](https://console.anthropic.com/)

填入后点击"测试"确认可用。

### 6.4 AI 模型配置

- **默认提供商**：选择辩论使用的主要 AI
- **max_tokens**：建议 2000-4000（太小会截断回答）
- **temperature**：建议 0.3-0.5（太低过于保守，太高过于激进）
- **模型名**：如 `deepseek-chat` / `gpt-4o-mini` / `claude-3-5-sonnet`

⚠️ 小白阶段先用默认值，资深篇再细调。

### 6.5 决策调优台（先了解）

| 配置项 | 含义 | 默认值 |
|--------|------|--------|
| 技术部权重 | 技术分析 Agent 的话语权 | 0.35 |
| 资金部权重 | 资金面 Agent 的话语权 | 0.35 |
| 新闻部权重 | 情绪面 Agent 的话语权 | 0.30 |
| 最低方向优势 | 多空观点差异低于此值 → 观望 | 见配置 |
| 最低数据质量 | 低于此等级不交易 | good |

小白阶段保持默认即可。

## 第 7 章 常见基础问题

### Q1：登录后 4 个卡片都是 0？

正常。新账户没有资金，需要启动模拟盘（见 02 篇）才有虚拟资金。

### Q2：行情页显示"数据加载中"很久？

- 检查后端容器是否正常运行：`docker compose ps`
- 检查后端日志：`docker compose logs --tail 100 backend`
- 第一次启动时，采集器需要 1-2 分钟预热

### Q3：AI 分析页面报错"AI 服务不可用"？

- 检查是否配置了 AI 提供商 API Key（设置页 → API 密钥管理）
- 检查代理配置（如果用 OpenAI/Claude）
- 检查 AI 模型配置（设置页 → AI 模型配置）

### Q4：忘记密码？

```powershell
# 进入数据库
docker exec -it moneyrobert-pro-db psql -U postgres -d moneyrobert

# 查看用户表
SELECT id, username, email FROM users;

# 退出
\q
```

⚠️ 直接改密码涉及加密，建议新建账户。或者参考 03 篇数据库运维章节。

### Q5：如何切换到生产环境？

修改 `docker-compose.yml`：

```yaml
backend:
  environment:
    - APP_SERVER__ENVIRONMENT=production
    - APP_SERVER__DEBUG=false
    - APP_SECURITY__SECRET_KEY=改成强随机字符串
```

然后 `docker compose up -d backend`。

### Q6：Windows 下 PowerShell 命令报错？

确保用 PowerShell（不是 CMD）。`&&` 在 PowerShell 5 不支持，改用 `;`：

```powershell
# 错误：cd D:\xxx && docker compose up
# 正确：
cd D:\xxx; docker compose up -d
```

或者使用项目目录作为 cwd 直接调用。

### Q7：浏览器控制台报 CORS 错误？

检查 `docker-compose.yml` 中 `APP_CORS__ORIGINS` 是否包含前端地址（默认已配置 `http://localhost:3000`）。
